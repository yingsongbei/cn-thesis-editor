# cn-thesis-editor

面向植物农业方向和更广泛生命科学领域的中文研究生学位论文语言编辑 skill，适用于博士和硕士学位论文。默认编辑标准仍按高质量博士学位论文要求执行。

## 适用场景

- 中文摘要、英文摘要
- 绪论和文献综述
- 材料与方法
- 结果
- 讨论
- 结论、创新点、不足与展望
- 参考文献和文内引用格式检查

## 运行环境

- 这是一个纯 Markdown Codex skill，不需要安装 Python、R、Node.js 或其他额外依赖。
- 推荐放置位置：`~/.codex/skills/cn-thesis-editor/`。
- 必需文件：`SKILL.md`。
- 可选文件：`README.md` 用于用户阅读说明，`agents/openai.yaml` 用于 Codex 界面显示。
- 适用于 Codex/支持本地 skills 的工作流。若在其他工具中使用，可直接复制 `SKILL.md` 的内容作为系统化写作规则。

## 默认原则

- 默认润色强度：中度
- 默认风格：中国顶尖大学优秀博士论文风格；用于硕士论文时，也默认按博士论文的严谨度、规范性和逻辑要求处理
- 学科倾向：植物农业方向，同时覆盖生命科学
- 允许拆分长句和调整段落顺序
- 更稳、更保守、更少主观判断
- 不允许新增事实、数据、机制、引用或参考文献信息
- 只有在用户明确允许增加背景或文献时，才可以新增背景和引用
- 新增文献必须多源核查真实性，尤其逐字核对 DOI
- 每次输出风险提示
- 语言要降低模板化和 AI 味，但不能牺牲科学严谨、正式和逻辑性
- 不以规避查重或 AI 检测为目标，也不承诺任何检测比例；通过规范引用、独立归纳、贴合真实研究内容和减少模板化表达来提高原创性与学术可信度

## 学术合规原则

毕业论文以稳妥、规范、可答辩为第一目标。skill 不会帮助规避查重或 AI 检测，也不会承诺把查重率或 AI 率降到某个数值。

它会做的是：

- 标出可能存在重复风险或改写不足的句子。
- 提醒哪些观点需要引用文献。
- 将文献综述从贴近原文的转述改为围绕论文主题的独立归纳。
- 保留必要术语、方法名称、基因名、物种名和经典概念，不做机械同义替换。
- 去掉空泛套话、模板化连接和 AI 式总结，让表达更具体、更贴合作者自己的研究。
- 在风险提示中说明哪些地方需要作者补充来源、核实事实或确认是否改变含义。

## 生命科学排版习惯

用户可以选择三种模式：

默认模式：只强制检查高确定性的规则，并提示不确定情况。默认使用此模式。

严格命名模式：在默认模式基础上，额外执行更严格的生物命名规则，例如限制性内切酶来源前缀斜体。

学校模板模式：完全按用户提供的学校、学院或导师模板执行。

默认模式下的主要规则：

- 物种拉丁名斜体，如 *Arabidopsis thaliana*、*Oryza sativa*、*Escherichia coli*。
- 属名首字母大写，种加词小写；首次出现写全名，后文可在无歧义时缩写为 *A. thaliana*。
- 植物基因符号斜体，如 *OsNAC1*、*AtWRKY33*；蛋白名和蛋白符号默认不斜体。
- 常见方法、数据库、软件、技术和试剂名不因英文而斜体，如 BLAST、KEGG、GO、qRT-PCR、RNA-seq、ChIP-seq、CRISPR/Cas9。
- 限制性内切酶默认保持标准连写，如 EcoRI，不写作 EcoR I；严格命名模式下可按要求写成 *Eco*RI。
- 常见拉丁表达如 in vivo、in vitro、in situ、ex vivo、in silico、et al.、de novo 默认不强制斜体；若学校模板要求斜体，则按模板统一处理。

## 润色强度

轻度：只修改病句、错别字、标点、不顺和轻微口语化表达，基本保留原句结构。

中度：优化清晰度、逻辑衔接、段落流动、术语一致性和学术表达；可以拆分长句，必要时调整句序或段落顺序。

重度：重组段落、合并重复内容、强化章节逻辑和论证层次；仍然不得改变科学含义，并需要标出所有可能影响含义的修改。

## 推荐用法

```text
请使用 cn-thesis-editor 对下面这段博士论文讨论部分进行中度润色，保持科学含义不变，并给出风险提示。
```

```text
请使用 cn-thesis-editor 修改英文摘要，大体和中文摘要逐句/意群对应，不需要逐字翻译，但要保持科学事实一致，语言逻辑接近 CNS 顶刊摘要风格。
```

```text
请使用 cn-thesis-editor 检查这段参考文献是否符合学校要求；如果没有学校要求，请按 GB/T 7714-2015 检查。
```

```text
请使用 cn-thesis-editor 在绪论中适当增加背景文献；新增文献必须核查真实性和 DOI，并在风险提示里说明核查情况。
```

```text
请使用 cn-thesis-editor 按严格命名模式检查这段文字中的物种拉丁名、基因名、蛋白名和限制性内切酶格式。
```

## 参考文献规则

学校或学院格式要求优先。没有学校要求时，默认按 GB/T 7714-2015 检查参考文献。GB/T 7714-2025 已发布，计划于 2026-07-01 实施；若学校明确采用 GB/T 7714-2025，则按 2025 版执行。

skill 不会虚构 DOI、页码、作者、年份、期刊名、网址或其他缺失字段，只会标出需要你核对的位置。

如果你明确允许增加背景文献，skill 应先通过出版社页面、DOI.org、Crossref、PubMed、Web of Science、Google Scholar 或期刊官网等可靠来源交叉核查；无法确认真实性或 DOI 的文献不能作为已确认引用加入正文。
